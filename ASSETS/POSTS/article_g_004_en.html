${S(8)}I've been working on a JavaScript program that explores in ordinals just like Rgetar's "Ordinal Explorer". My program is weaker compared to his because it can express less ordinals. And I use some Ordinal Notations base on some "official" OCFs (Ordinal Collapsing Functions) like Buchholz's OCF${C(1)} and Rathjen's OCF based on the "Mahlo Cardinal"(\( M \))${C(2)}. Buchholz's OCF is kinda easy and it didn't take long to complete, including the extended version${C(3)}. But Rathjen's little psi was pretty PAINFUL and took me a long time because the notation was so complicated and I had too many bugs to fix.

${S(8)}But because of the complexity of Rathjen's little psi, I think now is the time to share the "Fundamental Sequence System" (Or whatever this is called by this community) I came up with based on this OCF and how I managed to do this.

${S(8)}To my understanding, Fundamental Sequence Systems are like programs written in mathematical language that describes how ordinals works. Examples are this, this, and so on.

${S(8)}Fundamental Sequence Systems are NOT EASY to write. It's possible that some bugs and errors may occur and the System might not work as intended. But since the system I'm going to share is a byproduct of my well-tested JavaScript program, I think it's less likely that this has bugs.

Notation

${S(8)}The first step is to declare the symbols we use. Rathjen's little psi contains the following symbols:

${S(8)}Constant "\( \overline{0} \)" and "\( M \)",

${S(8)}Function symbol "\( \varphi \)", "\( \Phi \)", "\( \chi \)" and "\( \psi \)",

${S(8)}And other symbols "\( + \)", "\( ( \)", "\( ) \)", and "\( , \)".


${S(8)}That's a LOT symbols. Note that I added an overline to \( 0 \), this is because ordinals and ordinal notations are different, just like numbers and number strings. After all, in most cases, errors may happen if data types are messed up.

${S(8)}Now, we define a set \( S \) that contains all the strings composed of these 10 symbols. Technically, you can think of it as a function that takes a natural number less than the length of the string (The number of symbols the string uses) as input and returns a natural number less than 10. This is notated as \( s:L\rightarrow10 \), where \( L \) is the length of the string, which is a natural number.

${S(8)}And then, we define the set \( T_m \) (Or maybe \( T \) elsewhere) that contains all the valid strings and the set \( T_s \) (Or maybe \( PT \) elsewhere) that contains all the valid strings composed of only one terms. Apparently, \( S\supset T_m\supset T_s \) (Proper subset).

${S(8)}01. \( \overline{0}\in T_m \)

${S(8)}02. \( M\in T_m \) and \( M\in T_s \)

${S(8)}03. For all \( a\in T_s \) and \( b\in T_m\setminus\{\overline{0}\} \), \( a+b\in T_m \)

${S(8)}04. For all \( a\in T_m \) and \( b\in T_m \), \( \varphi(a,b)\in T_m \) and \( \varphi(a,b)\in T_s \)

${S(8)}05. For all \( a\in T_m \) and \( b\in T_m \), \( \Phi(a,b)\in T_m \) and \( \Phi(a,b)\in T_s \)

${S(8)}06. For all \( a\in T_m \) and \( b\in T_m \), \( \chi(a,b)\in T_m \) and \( \chi(a,b)\in T_s \)

${S(8)}07. For all \( a\in T_m \) and \( b\in T_m \), \( \psi(a,b)\in T_m \) and \( \psi(a,b)\in T_s \)

Abbreviation

${S(8)}We don't actually want to write functions as \( \varphi(a,b) \), \( \Phi(a,b) \), \( \chi(a,b) \) and \( \psi(a,b) \) because that makes the symbol hard to read. We also don't want to write \( 1 \) as \( \varphi_\overline{0}(\overline{0}) \) and \( \omega \) as \( \varphi_\overline{0}(\varphi_\overline{0}(\overline{0})) \) for the same reason. So, we gotta introduce some abbreviations.

${S(8)}01. \( \varphi(a,b)=\varphi_a(b) \)

${S(8)}02. \( \Phi(a,b)=\Phi_a(b) \)

${S(8)}03. \( \chi(a,b)=\chi_a(b) \)

${S(8)}04. \( \psi(a,b)=\psi_a(b) \)

${S(8)}05. \( \varphi_\overline{0}(\overline{0})=\overline{1} \)

${S(8)}06. \( \varphi_\overline{0}(\overline{1})=\overline{\omega} \)


${S(8)}Note that I added an overline to \( 1 \) and \( \omega \) as well for the same reason above.

Ordering

${S(8)}This is the most important part for ALL OCFs and ordinal notation systems. This is because without order, or comparison, there are no ordinals. What we're going to do here is just to define the less than symbol \( < \) because we can define all the other comparison symbols using only this one.

${S(8)}We define the binary relation \( < \) recursively as follows:

${S(8)}\( (a\in T_m,b\in T_m)\mapsto a<b\in\{\mathbb{T},\mathbb{F}\} \) (Which is written like \( {T_m}^2\rightarrow\{\mathbb{T},\mathbb{F}\}\quad(a,b)\mapsto a<b \) in other articles)


${S(8)}01. If \( b=\overline{0} \), return \( \mathbb{F} \)

${S(8)}02. If \( a=\overline{0}\land b\neq\overline{0} \), return \( \mathbb{T} \) (\( \land \) means "and")


${S(8)}03. If \( a=a_0+a_1\land b=b_0+b_1\land a_0,b_0\in T_s\land a_1,b_1\in T_m\setminus\{\overline{0}\} \), then:

${S(8)}03.01. If \( a_0<b_0 \) (Meaning that \( a_0<b_0 \) returns \( \mathbb{T} \)), return \( \mathbb{T} \)

${S(8)}03.02. If \( \neg a_0<b_0 \) (Meaning that \( a_0<b_0 \) returns \( \mathbb{F} \)), return \( \neg b_0<a_0\land a_1<b_1 \) (\( \neg \) means "not")

${S(8)}04. If \( a=a_0+a_1\land b,a_0\in T_s\land a_1\in T_m\setminus\{\overline{0}\} \), return \( a_0<b \)

${S(8)}05. If \( b=b_0+b_1\land a,b_0\in T_s\land b_1\in T_m\setminus\{\overline{0}\} \), return \( \neg b_0<a \)


${S(8)}06. If \( a=M\land b=M \), return \( \mathbb{F} \)


${S(8)}07. If \( a=\varphi_{a_0}(a_1)\land b=\varphi_{b_0}(b_1)\land a_0,a_1,b_0,b_1\in T_m \), then:

${S(8)}07.01. If \( a_0<b_0 \), return \( a_1<b \)

${S(8)}07.02. If \( b_0<a_0 \), return \( a<b_1 \)

${S(8)}07.03. If \( a_0=_*b_0 \), return \( a_1<b_1 \)


${S(8)}08. If \( a=\Phi_{a_0}(a_1)\land b=\Phi_{b_0}(b_1)\land a_0,a_1,b_0,b_1\in T_m \), then:

${S(8)}08.01. If \( a_0<b_0 \), return \( a_1<b \)

${S(8)}08.02. If \( b_0<a_0 \), return \( a<b_1 \)

${S(8)}08.03. If \( a_0=_*b_0 \), return \( a_1<b_1 \)


${S(8)}09. If \( a=\chi_{a_0}(a_1)\land b=\chi_{b_0}(b_1)\land a_0,a_1,b_0,b_1\in T_m \), then:

${S(8)}09.01. If \( a=_*b_1 \), return \( \neg b_0<a_0\vee\neg {b_0}^*<a \) (\( \vee \) means "or")

${S(8)}09.02. If \( a\neq_*b_1 \), then:

${S(8)}09.02.01. If \( a_0<b_0 \), return \( a_1<b\land{a_0}^*<b \)

${S(8)}09.02.02. If \( b_0<a_0 \), return \( \neg b_1<a\vee\neg{b_0}^*<a \)

${S(8)}09.02.03. If \( a_0=_*b_0 \), return \( a_1<b_1 \)


${S(8)}10. If \( a=\psi_{a_0}(a_1)\land b=\psi_{b_0}(b_1)\land a_0,a_1,b_0,b_1\in T_m \), then:

${S(8)}10.01. If \( a_0<b_0 \), return \( a_0<b \)

${S(8)}10.02. If \( b_0<a_0 \), return \( a<b_0 \)

${S(8)}10.03. If \( a_0=_*b_0 \), return \( a_1<b_1 \)


${S(8)}11. If \( a=M\land b\neq M \), then:

${S(8)}11.01. If \( b=\varphi_{b_0}(b_1)\land b_0,b_1\in T_m \), return \( (a<b_0\vee b_1\neq\overline{0})\land(\neg b_0<a\vee a<b_1) \)

${S(8)}11.02. If \( b=\Phi_{b_0}(b_1)\land b_0,b_1\in T_m \), return \( (a<b_0\vee b_1\neq\overline{0})\land(\neg b_0<a\vee a<b_1) \)

${S(8)}11.03. If \( b=\chi_{b_0}(b_1)\land b_0,b_1\in T_m \), return \( \mathbb{F} \)

${S(8)}11.04. If \( b=\psi_{b_0}(b_1)\land b_0,b_1\in T_m \), return \( \mathbb{F} \)


${S(8)}12. If \( a\neq M\land b=M \), then:

${S(8)}12.01. If \( a=\varphi_{a_0}(a_1)\land a_0,a_1\in T_m \), return \( a_0<b\land a_1<b \)

${S(8)}12.02. If \( a=\Phi_{a_0}(a_1)\land a_0,a_1\in T_m \), return \( a_0<b\land a_1<b \)

${S(8)}12.03. If \( a=\chi_{a_0}(a_1)\land a_0,a_1\in T_m \), return \( \mathbb{T} \)

${S(8)}12.04. If \( a=\psi_{a_0}(a_1)\land a_0,a_1\in T_m \), return \( \mathbb{T} \)


${S(8)}13. If \( a=\varphi_{a_0}(a_1)\land(b=\Phi_{b_0}(b_1)\vee b=\chi_{b_0}(b_1)\vee b=\psi_{b_0}(b_1))\land a_0,a_1,b_0,b_1\in T_m \), then:

${S(8)}13.01. Return \( a_0<b\land a_1<b \)


${S(8)}14. If \( (a=\Phi_{a_0}(a_1)\vee a=\chi_{a_0}(a_1)\vee a=\psi_{a_0}(a_1))\land b=\varphi_{b_0}(b_1)\land a_0,a_1,b_0,b_1\in T_m \), then:

${S(8)}14.01. Return \( (a<b_0\vee b_1\neq\overline{0})\land(\neg b_0<a\vee a<b_1) \)


${S(8)}15. If \( a=\Phi_{a_0}(a_1)\land b=\chi_{b_0}(b_1)\land a_0,a_1,b_0,b_1\in T_m \), then:

${S(8)}15.01. If \( b_0=\overline{0} \), return \( a<b_1 \)

${S(8)}15.02. If \( b_0\neq\overline{0} \), return \( a_0<b\land a_1<b \)


${S(8)}16. If \( a=\chi_{a_0}(a_1)\land b=\Phi_{b_0}(b_1)\land a_0,a_1,b_0,b_1\in T_m \), then:

${S(8)}16.01. If \( a_0=\overline{0} \), return \( a_1<b \)

${S(8)}16.02. If \( a_0\neq\overline{0} \), then:

${S(8)}16.02.01. If \( a<b_0 \), return \( \mathbb{T} \)

${S(8)}16.02.02. If \( b_0<a \), return \( a<b_1 \)

${S(8)}16.02.03. If \( a=_*b_0 \), return \( b_1\neq\overline{0} \)


${S(8)}17. If \( a=\Phi_{a_0}(a_1)\land b=\psi_{b_0}(b_1)\land a_0,a_1,b_0,b_1\in T_m \), return \( a_0<b\land a_1<b\land a<b_0 \)


${S(8)}18. If \( a=\psi_{\chi_{a_0}(a_1)}(a_2)\land b=\Phi_{b_0}(b_1)\land a_0,a_1,a_2,b_0,b_1\in T_m \), then:

${S(8)}18.01. If \( a_0=\overline{0} \), return \( a_1<b \)

${S(8)}18.02. If \( a_0\neq\overline{0} \), then:

${S(8)}18.02.01. If \( a<b_0 \), return \( \mathbb{T} \)

${S(8)}18.02.02. If \( b_0<a \), return \( a<b_1 \)

${S(8)}18.02.03. If \( a=_*b_0 \), return \( b_1\neq\overline{0} \)


${S(8)}19. If \( a=\chi_{a_0}(a_1)\land b=\psi_{b_0}(b_1)\land a_0,a_1,b_0,b_1\in T_m \), return \( a<b_0\land a\in C(b_0,b_1) \)


${S(8)}20. If \( a=\psi_{\chi_{a_0}(a_1)}(a_2)\land b=\chi_{b_0}(b_1)\land a_0,a_1,a_2,b_0,b_1\in T_m \), then:

${S(8)}20.01. If \( a=_*b_1 \), return \( \neg b_0<a_0\vee\neg{b_0}^*<\chi_{a_0}(a_1)\vee\neg b_0\in C(\chi_{a_0}(a_1),a_2) \)

${S(8)}20.02. If \( a\neq_*b_1 \), return \( \neg b<\chi_{a_0}(a_1)\vee\neg b\in C(\chi_{a_0}(a_1),a_2) \)


${S(8)}Wow! That's a LONG definition!!! This is the reason why Rathjen's little psi is difficult. Most of these cases are derived from Rathjen's PDF on this OCF${C(2)}. ${S(8)}And now, we can define all the other comparison symbols:

${S(8)}\( (a\in T_m,b\in T_m)\mapsto a>b\in\{\mathbb{T},\mathbb{F}\} \)

${S(8)}return \( b<a \)


${S(8)}\( (a\in T_m,b\in T_m)\mapsto a\le b\in\{\mathbb{T},\mathbb{F}\} \)

${S(8)}return \( \neg b<a \)


${S(8)}\( (a\in T_m,b\in T_m)\mapsto a\ge b\in\{\mathbb{T},\mathbb{F}\} \)

${S(8)}return \( \neg a<b \)


${S(8)}\( (a\in T_m,b\in T_m)\mapsto a=_*b\in\{\mathbb{T},\mathbb{F}\} \)

${S(8)}\( (a\in T_m,b\in T_m)\mapsto a\neq_*b\in\{\mathbb{T},\mathbb{F}\} \)

${S(8)}We can see that the special symbols \( =_* \) and \( \neq_* \) are introduced. The difference from \( = \) and \( \neq \) respectively is that \( = \) and \( \neq \) cares about the symbols we use and \( =_* \) and \( \neq_* \) only cares about ordinal values, for example:

${S(8)}\( \chi_{\overline{3}}(\overline{0})=\chi_{\overline{1}+\overline{1}+\overline{1}}(\overline{0}) \)

${S(8)}\( \chi_{\overline{2}}(\chi_{\overline{3}}(\overline{0}))=\chi_{\overline{1}+\overline{1}}(\chi_{\overline{1}+\overline{1}+\overline{1}}(\overline{0})) \)

${S(8)}\( \chi_{\overline{3}}(\overline{0})=_*\chi_{\overline{2}}(\chi_{\overline{3}}(\overline{0})) \) but \( \chi_{\overline{3}}(\overline{0})\neq\chi_{\overline{2}}(\chi_{\overline{3}}(\overline{0})) \)

Normal Form

${S(8)}We can also see some symbols like \( a^* \) and \( a\in C(b_0,b_1) \) appeared in the definition of \( a<b \). We will define them as well.

${S(8)}We define the symbol \( a^* \) as follows:

${S(8)}\( (a\in T_m)\mapsto a^*\in T_m \)

${S(8)}01. If \( a=\overline{0}\vee a=M \), return \( \overline{0} \)

${S(8)}02. If \( a=a_0+a_1\land a_0\in T_s\land a_1\in T_m\setminus\{\overline{0}\} \), then:

${S(8)}02.01. If \( a^*<b^* \), return \( b^* \)

${S(8)}02.02. If \( \neg a^*<b^* \), return \( a^* \)

${S(8)}03. If \( a=\varphi_{a_0}(a_1)\land a_0,a_1\in T_m \), then:

${S(8)}03.01. If \( a^*<b^* \), return \( b^* \)

${S(8)}03.02. If \( \neg a^*<b^* \), return \( a^* \)

${S(8)}04. If \( (a=\Phi_{a_0}(a_1)\vee a=\chi_{a_0}(a_1)\vee a=\psi_{a_0}(a_1))\land a_0,a_1\in T_m \), return \( a \)


${S(8)}We define addition \( Add(a,b) \) as follows. This is just to prevent inappropriate forms like \( 0+a \) and \( a+0 \). I didn't have to do this in my original program because \( a+b \) is just written as \( ab \) in my program and there is no multiplication.

${S(8)}\( (a\in T_m,b\in T_m)\mapsto Add(a,b)\in T_m \)

${S(8)}01. If \( a=\overline{0} \), return \( b \)

${S(8)}02. If \( a\neq\overline{0}\land b=\overline{0} \), return \( a \)

${S(8)}03. If \( a\neq\overline{0}\land b\neq\overline{0} \), return \( a+b \)


${S(8)}We define succession \( S(a) \) as follows. Which is simple but comes in handy in simplifying later definitions.

${S(8)}\( (a\in T_m)\mapsto S(a)\in T_m\setminus\{\overline{0}\} \)

${S(8)}return \( Add(a,\overline{1}) \)


${S(8)}We define the set \( R \). Which is actually \( (Reg\cap M)\setminus\chi_\overline{0}(\overline{0}) \)

${S(8)}01. For all \( a\in T_m \), \( \chi_a(\overline{0})\in R \)

${S(8)}02. For all \( a\in T_m \) and \( b\in T_m \), \( \chi_a(S(b))\in R \)


${S(8)}We define the set \( C(a,b) \). Which is actually the first step or the most important step in all OCF definitions.

${S(8)}01. \( \overline{0}\in C(a,b) \) and \( M\in C(a,b) \)

${S(8)}02. If \( a=\chi_{a_0}(\overline{0}) \), For all \( c\le a_0 \), \( c\in C(a,b) \)

${S(8)}03. If \( a=\chi_{a_0}(S(a_1)) \), For all \( c\le \chi_{a_0}(a_1) \), \( c\in C(a,b) \)

${S(8)}05. For all \( c_0,c_1\in C(a,b) \), \( \varphi_{c_0}(c_1)\in C(a,b) \)

${S(8)}06. For all \( c_0,c_1\in C(a,b) \) and \( c_0,c_1<M \) and \( c_0\neq\overline{0} \), \( \Phi_{c_0}(c_1)\in C(a,b) \)

${S(8)}07. For all \( c_0,c_1\in C(a,b) \), \( \chi_{c_0}(c_1)\in C(a,b) \)

${S(8)}08. For all \( c_0,c_1\in C(a,b) \) and \( c_1<b \) and \( c_1\in C(c_0,c_1) \), \( \psi_{c_0}(c_1)\in C(a,b) \)

${S(8)}09. For all \( d\in C(a,b)\cap R \) and \( c<d<a \), \( c\in C(a,b) \)


${S(8)}We define the set \( T \) (Or maybe \( OT \) elsewhere) that contains all the valid strings in NORMAL FORM.

${S(8)}01. \( \overline{0}\in T \) and \( M\in T \)

${S(8)}02. For all \( a_0,a_1,\cdots,a_n\in T_s \) and \( a_0\ge a_1\ge\cdots\ge a_n \), \( a_0+a_1+\cdots+a_n\in T \)

${S(8)}03. For all \( a_0,a_1<\varphi_{a_0}(a_1) \), \( \varphi_{a_0}(a_1)\in T \)

${S(8)}\( (a\in T_m)\mapsto Norm(a)\in T \)

${S(8)}02. If \( \neg a\in T \), then:

${S(8)}02.02.02. If \( a=a_0 \), return \( a_0 \)

${S(8)}02.03. If \( a=\Phi_{a_0}(a_1)\land a_0,a_1\in T_m \), then:

${S(8)}02.03.02. If \( a_0\neq\overline{0} \), then:

${S(8)}02.03.02.01. If \( a=a_1 \), return \( a_1 \)

${S(8)}02.03.02.02. If \( a=a_0 \), return \( a_0 \)

${S(8)}02.03.02.03. If \( a\neq a_0\land a\neq a_1 \), return \( a \)

${S(8)}02.04. If \( a=\chi_{a_0}(a_1)\land a_0,a_1\in T_m \), return \( a_1 \)

${S(8)}02.05. If \( a=\psi_{a_0}(a_1)\land a_0,a_1\in T_m \), return \( a \)

${S(8)}05. For all \( a_0,a_1\in T(M) \), \( \chi_{a_0}(a_1)\in T(M) \)

${S(8)}07. For all \( a\in T(M) \) and \( M<a \), \( \varphi_\overline{0}(a)\in T(M) \)


${S(8)}Apparently, the definition has made some restrictions on the use of function \( \varphi \) and \( \psi \). I'll explain the possible reasons later in this article.

Cofinality

${S(8)}Now, we define cofinality \( Cof \), which is important in defining the FS (Fundamental Sequence). Some people will use \( Dom \) instead, which stands for "Domain". I don't think this makes as many senses as \( Cof \). The only idea I can think of is to regard every ordinal as a function whose domain is its cofinality. Actually, this notation is due to Buchholz${C(1)} in his OCF.

${S(8)}\( (a\in T)\mapsto Cof(a)\in T \)

${S(8)}01. If \( a=\overline{0}\vee a=M \), return \( a \)

${S(8)}02. If \( a=a_0+a_1\land a_0\in T_s\land a_1\in T_m\setminus\{\overline{0}\} \), return \( Cof(a_1) \)

${S(8)}03.01. If \( \neg Cof(a_1)<\overline{\omega} \), return \( Cof(a_1) \)

${S(8)}03.02. If \( \neg Cof(a_0)<\overline{\omega}\land Cof(a_1)<\overline{\omega} \), return \( Cof(a_0) \)

${S(8)}03.04. If \( a_0=\overline{0}\land a_1=\overline{0} \), return \( \overline{1} \)

${S(8)}04.01. If \( a\in R \), return \( a \)

${S(8)}04.02. If \( \neg a\in R \), return \( Cof(a_1) \)

${S(8)}05.01. If \( \chi_{a_0}(a_1)\le Cof(a_2) \), return \( \overline{\omega} \)

${S(8)}05.02. If \( \overline{\omega}\le Cof(a_2)\land Cof(a_2)<\chi_{a_0}(a_1) \), return \( Cof(a_2) \)

${S(8)}05.03. If \( Cof(a_2)<\overline{\omega} \), then:

${S(8)}05.03.01. If \( Cof(a_0)<\overline{\omega} \), return \( \overline{\omega} \)

${S(8)}05.03.02. If \( Cof(a_0)=M \), return \( \overline{\omega} \)

${S(8)}05.03.03. If \( \overline{\omega}\le Cof(a_0)\land Cof(a_0)<M \), return \( Cof(a_0) \)

Fundamental Sequence

${S(8)}Now comes the most important part in googology. If we use an ordinal in the FGH (Fast-Growing Hierarchy) function, to make the result number well-defined, we need to define the FS for all the ordinals less than or equal to this ordinal.

${S(8)}But the problem is, mathematicians don't usually define a FS system for their OCFs (Although Buchholz DOES${C(1)}). This is because this process is completely artificial (At least for now). So, we have to do this ourselves and this is DIFFICULT.

${S(8)}And this is another long definition other than the \( < \) symbol. Wow. Line 07.01. explains why Rathjen made restrictions on the function \( \psi \), this is because when we plug something greater than \( M \) into the function, \( M \) actually acts like something smaller that looks like \( \psi_{\chi_{\Gamma_{M+1}}(0)}(0) \), which can be considered a bug. The reason why Rathjen also made restrictions on the function \( \varphi \) is probably because \( \psi_{\chi_{0}(0)}(\psi_{\chi_{\varphi_1(M+1)}(0)}(0))=\psi_\Omega(\psi_{\chi_{\varepsilon_{M+1}}(0)}(0)) \) is already equal to the PTO (Proof-Theoretic Ordinal) of \( KPM \).

${S(8)}Line 07.04.04.01.01. and 07.04.04.01.02. are the trickiest lines in the definition and took me about 8 hours to figure out. This is because it's extremely hard to reason the difference between the FS of \( \psi_{\chi_M(0)}(0) \) and the FS of \( \psi_{\chi_{\psi_{\chi_M(0)}(0)}(0)}(0) \) without using mathematical analysis and thus extremely hard to come up with the FS using programming language.

Conclusion

${S(8)}So, there you have it, the Fundamental Sequence System of Rathjen's little psi. I actually doubt whether this thing is done that thoroughly on the internet before this article is published. I think this OCF is very important for studying large ordinals, even though there are even stronger OCFs. I think after defining such system, some brief analysis is required, but since I've done this in the past, you can check out my previous articles for the analysis.

${S(8)}The "Small Rathjen Ordinal" (SRO), which is equal to \( \psi_\Omega(\psi_{\chi_{\varepsilon_{M+1}}(0)}(0)) \), is the PTO of \( KPM \). Now, with the definitions above, I think I can safely say that \( f_{SRO}(10) \) is well-defined and will most likely surpass about 85% of the large numbers ever coined on the internet (Mar 23, 2024). (Although technically, we have to convert natural numbers to symbols before plugging into the FGH function.)

So, I think writing a program is the best way to make the definition robust. Because it's hard to know whether there are errors in the definition.

${S(8)}All that being said, I think it's still possible that the definition in this article may contain bugs, either from this article, or from my original program itself. So, if I'm notified that there are bugs, I'll fix'em asap.
