${S(8)}Church-Kleene Ordinal \( \omega_1^{CK} \) is the first non-recursive, i.e. uncomputable ordinal. This ordinal is so large that even computers cannot "handle" and thus very important and known to a lot of googologists. Since the FGH (Fast Growing Hierarchy) function uses extremely large ordinals to represent extremely large numbers, I believe a lot of googologists wonder how fast-growing the function \( f_{\omega_1^{CK}}(n) \) is.

${S(8)}Just like other large ordinals, to make the FGH function work properly, we have to define a system of FS (Fundamental Sequence) for all the ordinals less than or equal to this ordinal. And defining such FS requires defining a "code" for each computable function.

${S(8)}In this article, I will define such encoding using "Lambda Calculus" and "SK Combinatory Logic" to find the approximation between the function \( f_{\omega_1^{CK}}(n) \) under this encoding and the Busy Beaver Function \( \Sigma(n) \). I'm not going to explain all of these concepts in depth and will jump right into the most important concepts for this article instead. If you want to learn more about Lambda Calculus and SK Combinatory Logic, check out this PDF file and it will help a lot.

How does \( \omega_1^{CK} \) work?

${S(8)}\( \omega_1^{CK} \) uses an ordinal system called "Kleene's \( \mathcal{O} \)". Every element in this system is a natural number called "notation" assigned to a recursive, i.e. computable ordinal. For example:

${S(8)}\( {0}_\mathcal{O}=0 \)

${S(8)}\( {1}_\mathcal{O}=1 \)

${S(8)}\( {7}_\mathcal{O}=\omega \)

${S(8)}\( {69}_\mathcal{O}=\varepsilon_0 \)

${S(8)}\( {420}_\mathcal{O}=\Gamma_0 \)

${S(8)}\( {114514}_\mathcal{O}=BHO=\psi_0(\Omega_2) \)

${S(8)}\( {7355608}_\mathcal{O}=LBO=\psi_0^\ast(\Omega_{\Omega_{\Omega_\ddots}}) \)

${S(8)}\( {({10}^{100})}_\mathcal{O}=LRO=\Psi_\Omega^0(\varepsilon_{K+1}) \)


${S(8)}And then, we need to define an encoding that assigns every computable function that takes ONE natural number as input and returns ONE natural number with a natural number I call "code". (There are countably many computer codes and therefore countably many computable functions, which is the reason why the natural numbers are used) For example:

${S(8)}\( F_0(n)=n+1 \)

${S(8)}\( F_1(n)=2n \)

${S(8)}\( F_2(n)=n! \)

${S(8)}\( F_3(n)=7777777\uparrow\uparrow\uparrow\uparrow\uparrow\uparrow\uparrow n \)

${S(8)}\( F_4(n)=TREE(n) \)


${S(8)}The ordinal system is defined as follows:

${S(8)}1. Base Rule:

${S(8)}\( 0\in\mathcal{O}\land 0_\mathcal{O}=0 \)


${S(8)}2. Successor Rule:

${S(8)}\( n\in\mathcal{O}\land n_\mathcal{O}=\nu\Rightarrow 2^n\in\mathcal{O}\land{(2^n)}_\mathcal{O}=\nu+1\land n<_\mathcal{O}2^n \)


${S(8)}3. Limit Rule:

${S(8)}\( \forall n\in\omega(F_m(n)\in\mathcal{O}\land F_m(n)<_\mathcal{O}F_m(n+1))\Rightarrow \)

${S(8)}\( 3*5^m\in\mathcal{O}\land{(3*5^m)}_\mathcal{O}=sup\{k:k=F_m(n)\land n\in\omega\}\land\forall n\in\omega(F_m(n)<_\mathcal{O}3*5^m) \)


${S(8)}Where \( \mathcal{O} \) is the set of all notations, \( n_\mathcal{O} \) is the operation that converts notations into their ordinals and \( <_\mathcal{O} \) is the partial ordering of this system.


${S(8)}And the FS of \( \omega_1^{CK} \) is defined as follows:

${S(8)}\( a_0=0 \)

${S(8)}\( a_{n+1}=min\{b:b\in\mathcal{O}\land a_n<b\land a_n<_\mathcal{O}b\} \)

${S(8)}\( \omega_1^{CK}[n]={(a_n)}_\mathcal{O} \)


${S(8)}For ordinals \( \alpha<\omega_1^{CK} \):

${S(8)}\( 0[n]=0 \)

${S(8)}\( (\alpha+1)[n]=\alpha \)

${S(8)}\( \alpha[n]={F_{min\{m:{(3*5^m)}_\mathcal{O}=\alpha\}}(n)}_\mathcal{O} \) if \( \alpha \) is a limit ordinal


${S(8)}I wonder why the notation is \( 2^n \) for successor ordinals and \( 3*5^m \) for limit ordinals. I think this is unnecessary to avoid repetition and makes the notations grow too fast. Encodings like this that use prime numbers and exponentiation are commonly used in mathematics. I think the notation of \( 2n \) for successor ordinals and \( 2m+1 \) for limit ordinals is sufficient. But since this does have some advantages for later definitions, we will focus on the original definition in this article.

${S(8)}So, it seems that we have already defined the FS of this ordinal. But wait! In this definition, the encoding for all computable functions is UNSPECIFIED. This is because the limit ordinal of this ordinal system is the same for all encodings. (Which is something I'm unable to prove or disprove)

${S(8)}Since the output of \( f_{\omega_1^{CK}}(n) \) depends on the FS and therefore the encoding we use, we have to define the encoding ourselves. But how do we do this? What does "computable" even mean? What are the models of computation?

${S(8)}It turns out there are 2 well-known models of computation. One is the "Turing Machine", and the other is "Lambda Calculus". They're proven to "do the same thing" by "Church-Turing Thesis". Since using Turing Machines isn't as intuitive and convenient compared to using Lambda Calculus, we use Lambda Calculus to define the encoding in this article.

The Lambda Calculus

${S(8)}Lambda Calculus is a formal system for expressing computation invented by Alonzo Church. Yes. The "Church" in the name "Church-Kleene Ordinal". If you're familiar with programming languages like Python, you can understand it as a system of anonymous functions. The composition of Lambda Terms in this system consists of the following 3 rules:

${S(8)}1. Variable Rule: A variable is a Lambda Term. For example: \( x_0,x_1,x_2\cdots \)

${S(8)}2. Abstraction Rule: If \( x \) is a variable and \( M \) is a Lambda Term, then \( \lambda x.M \) is a Lambda Term, which simply means to define a function.

${S(8)}3. Application Rule: If both \( M \) and \( N \) are Lambda Terms, then \( (MN) \) is a Lambda Term, which simply means to plug something into the function.


${S(8)}The following are examples of Lambda Terms. For the sake of readability, the terms are considered left-associative and the leftmost parentheses are omitted. For example: \( ((AB)C)=(AB)C=ABC \). Furthermore, multiple lambda abstractions are contracted. For example: \( \lambda x.\lambda y.x=\lambda xy.x \).

${S(8)}\( I=\lambda x.x \)

${S(8)}\( True=\lambda xy.x \)

${S(8)}\( False=\lambda xy.y \)

${S(8)}\( And=\lambda xy.xyx \)

${S(8)}\( Or=\lambda xy.xxy \)

${S(8)}\( 2_C=\lambda fx.f(fx) \)


${S(8)}The following are the 3 ways to evaluate every Lambda Term. They use Greek Alphabets, which looks kinda mythical to me. Why do they use such alphabets?

${S(8)}1. \( \alpha \)-conversion, which just changes names bound variables.

${S(8)}2. \( \beta \)-reduction, which just plugs inputs into functions.

${S(8)}3. \( \eta \)-reduction, which simplifies functions in very complicated ways. The simplest of which is \( \lambda x.Mx=_\eta M \), where \( x \) isn't a free variable of \( M \)


${S(8)}For example:

${S(8)}\( (\lambda x.x+1)4=_\beta 4+1=5 \)

${S(8)}\( \lambda xy.xxy=\lambda x.\lambda y.(xx)y=_\eta \lambda x.xx \)


${S(8)}You will sometimes see notations like \( \lambda x.x+1 \) and \( \lambda\alpha.\omega^\alpha \), which simply means \( f(n)=n+1 \) and \( f(\alpha)=\omega^\alpha \). This is because EVERY LAMBDA TERM IS A FUNCTION. But these notations look weird (At least for now). How to write \( n+1 \) and \( \omega^\alpha \) in Lambda Terms? We have to define everything like Booleans, numbers, operations, arrays and so on using Lambda Terms.

Defining Numbers And Operations Using Lambda Calculus

${S(8)}So, how to define numbers and operations using Lambda Calculus? Just like defining natural numbers in set theory, there are a lot of ways to do this. The most common of them is using Church Numerals:

${S(8)}\( 0_C=\lambda fx.x=_\alpha \lambda xy.y=False \)

${S(8)}\( 1_C=\lambda fx.fx=_\eta \lambda x.x=I \)

${S(8)}\( 2_C=\lambda fx.f(fx) \)

${S(8)}\( 3_C=\lambda fx.f(f(fx)) \)

${S(8)}\( 4_C=\lambda fx.f(f(f(fx))) \)


${S(8)}It's just applying a function \( f \) on \( x \) multiple times, which is really neat. Note that 0 and 1 are equivalent to something else, which can sometimes be confusing. Actually, there are a lot of such cases in Lambda Calculus.

${S(8)}The basic operations on numbers are defined as follows:

${S(8)}Succession: \( Suc(n)=\lambda nfx.f(nfx) \)

${S(8)}Addition: \( a+b=\lambda abfx.af(bfx) \)

${S(8)}Multiplication: \( a*b=\lambda abf.a(bf) \)

${S(8)}Exponentiation: \( a^b=\lambda ab.ba \)


${S(8)}You can check whether these definitions are correct. I think the exponentiation function is pretty neat because you just swap Lambda Terms. Now, we can define all the big numbers in this way. We can also define subtraction, division and so on, but the definition is complicated and not important for this article. You can check out the PDF above if you want to know more.

${S(8)}Remember, every Lambda Term is a function, which means all of the Church Numerals defined above are functions. For example, \( 3_C \) not only represents the natural number 3, but also represents the function \( f(n)=n^3 \).

The SK Combinatory Logic

${S(8)}We might want to convert Lambda Terms into codes right away, but we have a problem. Due to \( \alpha \)-conversion, we can rename the variables in Lambda Terms whatever we want and generate multiple codes:

${S(8)}\( True=\lambda xy.x=_\alpha\lambda ab.a=_\alpha\lambda\mu\nu.\mu=_\alpha\lambda\textrm{Æ}\textrm{Ø}.\textrm{Æ}=_\alpha\lambda\textrm{甲}\textrm{乙}.\textrm{甲}=_\alpha\lambda\textrm{あ}\textrm{い}.\textrm{あ} \)


${S(8)}However, there's a solution. We can plug a function into a function and get something that works the same as a function familiar to us:

${S(8)}\( True\ I\ a\ b=(\lambda xy.x)I\ a\ b=_\beta(\lambda y.I)a\ b=_\beta I\ b=(\lambda x.x)b=_\beta b \)

${S(8)}\( False\ a\ b=(\lambda xy.y)a\ b=_\beta(\lambda y.y)b=_\beta b \)

${S(8)}\( True\ I\ a\ b=False\ a\ b=b \)

${S(8)}\( True\ I=False \)


${S(8)}So, we can just combine functions without using variables. And this is the core of Combinatory Logic. The following is a list of Combinators, which are Lambda Terms that can be combined into other functions. They're sometimes named like birds, which is interesting:

${S(8)}Kestrel: \( K=\lambda xy.x=True \)

${S(8)}Starling: \( S=\lambda abc.ac(bc) \)

${S(8)}Ibis: \( I=\lambda x.x=1_C \)

${S(8)}Kite: \( KI=\lambda xy.y=False=0_C \)

${S(8)}Mockingbird: \( M=\lambda x.xx=Or \)

${S(8)}Cardinal (Yes. This is also a bird name.): \( C=\lambda abc.acb \)

${S(8)}Bluebird: \( B=\lambda abc.a(bc) \)

${S(8)}Thrush: \( T=\lambda ab.ba \)

${S(8)}Vireo: \( V=\lambda abc.cab \)


${S(8)}They act like "atoms" of Lambda Terms, forming the whole "Lambda Universe". The question is, how many of them do we need to construct every Lambda Term? It turns out that we only need 2 of them, which are Kestrel and Starling.

${S(8)}If 2 of them can create any Lambda Terms, it means we can convert every Lambda Term into only Kestrels and Starlings, or SK form. And this is the core of SK Combinatory Logic. Sometimes it's known as the SKI Combinatory Logic because Ibis is also used, but since \( I=SKK \), we don't need Ibis to be an "atom combinator", although it can still be used as a useful abbreviation.

${S(8)}By the way, I should mention that there is also the "Iota Combinator", which is defined as: \( \iota=\lambda x.xSK \). And also:

${S(8)}\( I=\iota\iota \)

${S(8)}\( K=\iota(\iota(\iota\iota)) \)

${S(8)}\( S=\iota(\iota(\iota(\iota\iota))) \)


${S(8)}This enables us to write every Lambda Term using only Iota Combinators, which is really neat. The PROBLEM is, this form uses much more combinators than SK form. It's important to minimize the code for later purposes. Therefore, we must use as few combinators as possible, which ultimately makes this form not worth it. So, in this article, we focus on the SK form.

${S(8)}So, how do we convert every Lambda Term into SK form? The conversion \( Con \) is defined as follows:

${S(8)}\( Con[x]=x \), \( x \) is a variable.

${S(8)}\( Con[AB]=Con[A]\ Con[B] \)

${S(8)}\( Con[\lambda x.x]=SKK \)

${S(8)}\( Con[\lambda x.M]=K\ Con[M] \), \( x \) isn't a free variable of \( M \)

${S(8)}\( Con[\lambda x.MN]=S\ Con[\lambda x.M]\ Con[\lambda x.N] \)

${S(8)}\( Con[\lambda x.Mx]=M \), \( x \) isn't a free variable of \( M \)


${S(8)}For example, let's try to convert \( 3_C \) into SK form:

${S(8)}\( \begin{align} & 3 \\ = & \lambda fx.f(f(fx)) \\ = & \lambda f.[\lambda x.f(f(fx))] \\ = & \lambda f.S(\lambda x.f)(\lambda x.f(fx)) \\ = & \lambda f.S(Kf)(\lambda x.f(fx)) \\ = & S(\lambda f.S(Kf))(\lambda fx.f(fx)) \\ = & S(S(\lambda f.S)(\lambda f.Kf))(2) \\ = & S(S(KS)K)(2) \end{align} \)


${S(8)}We stop right here, because guess what? We've just converted the succession function into SK form:

${S(8)}\( Suc=S(S(KS)K) \)


${S(8)}We can just proceed with \( 3=Suc(2)=Suc(Suc(1))=Suc(Suc(Suc(0))) \). Now, we have to convert \( 0_C \) into SK form:

${S(8)}\( 0_C=False=KI=K(SKK) \)


${S(8)}But actually, there's a better way to represent \( 0_C \):

${S(8)}\( SKab=Kb(ab)=b \):

${S(8)}\( K(SKK)ab=(SKK)b=Ib=b \):

${S(8)}\( SK=K(SKK) \):


${S(8)}Which is a little optimization that is not easy to find because computers cannot determine whether 2 codes do the same thing, which requires to solve the "Halting Problem", which is known to be UNDECIDABLE. Furthermore:

${S(8)}\( 1_C=I=SKK \)


${S(8)}If we just plug \( 0_C \) into the succession function, we get a Lambda Term with 7 symbols, which uses more combinators. Remember, we must use as few combinators as possible.

${S(8)}Now, we can define Church Numerals in the SK form:

${S(8)}\( 0_C=SK \)

${S(8)}\( 1_C=SKK \)

${S(8)}\( (n+2)_C=S(S(KS)K)(n+1)_C,n\in\omega \)

Back To The Encoding

${S(8)}We're finally ready to define an encoding for all computable functions. Since Lambda Calculus expresses computation, every Lambda Term is computable. And since every Lambda Term is a function, the functions we're going to encode are all of the Lambda Terms. And since we can convert every Lambda Term into its SK form, the codes are all based on the SK form of the Lambda Term.

${S(8)}Firstly, we define the length of Lambda Term in SK form:

${S(8)}\( Len[K]=Len[S]=1 \)

${S(8)}\( Len[AB]=Len[A]+Len[B] \)


${S(8)}So, how many Lambda Terms are there that has a length of 1? The answer is 2.

${S(8)}What about Lambda Terms whose length is 2 (L2 Lambda Terms)? Since we have to combine two L1 Lambda Terms, we have 2*2=4.

${S(8)}What about L3? We can combine a L1 Lambda Term and a L2 Lambda Term in two ways. So, we get 2*4+4*2=16.

${S(8)}What about L4? We can combine a L1 Lambda Term and a L3 Lambda Term in two ways. We can also combine two L2 Lambda Terms, so the answer is 2*16+4*4+16*2=80

${S(8)}And we have 448 L5 Lambda Terms, 2,688 L6 Lambda Terms, 16,896 L7 Lambda Terms and so on. What's the pattern of the sequence? We can't just calculate them artificially. It turns out that there is a formula for all these numbers. Let A(n) be the amount of Ln Lambda Terms, we have:

${S(8)}\( L(n)=2^n*C(n-1),n\in\omega\backslash\{0\} \)

${S(8)}\( C(n)=\frac{2n!}{n!(n+1)!},n\in\omega \)


${S(8)}Where C(n) is the "Catalan Sequence", which is an interesting and important sequence that doesn't grow that fast in googological sense and can be bounded by \( 4^n \) (Which is not hard to prove using mathematical induction), which makes L(n) bounded by \( 8^n \).

${S(8)}Now we can finally define the encoding function \( N \) and the function \( F_a \) as follows:

${S(8)}\( N[K]=0,N[S]=1 \)

${S(8)}\( \begin{align} N[AB] = & \sum_{n=1}^{Len[AB]-1}{L(n)}+\sum_{n=1}^{Len[A]-1}{L(n)*L(Len[AB]-n)} \\ + & (N[A]-\sum_{n=1}^{Len[A]-1}{L(n)})*L(Len[B])+N[B]-\sum_{n=1}^{Len[B]-1}{L(n)}) \end{align} \)

${S(8)}\( F_a=A\Leftrightarrow N[A]=a \)


${S(8)}Woah. This function looks terrifying! But rest assured, this function can encode all the Lambda Terms without repetitions and vacancies. I call this encoding the Optimized Combinatory Logic Encoding (OCLE for short). In fact, there is a way to convert every Lambda Term into a binary code${C(1)}, which can immediately generate another encoding function that is relatively simple. But sometimes it can result in some invalid codes and we want the code to be optimized to be used as a standard.

${S(8)}Let's use \( Suc=S(S(KS)K) \), the successor function as an example. After plugging it into the formula, we get \( N[S(S(KS)K)]=240 \), which means:

${S(8)}\( F_{240}=S(S(KS)K)\Rightarrow F_{240}(n)=n+1 \).


${S(8)}Moreover, this is a function that takes ONE Church Numeral as input and returns ONE Church Numeral, which is what we need in the first place because we need computable functions that takes ONE natural number as input and returns ONE natural number and Church Numerals represent natural numbers.

${S(8)}But there's a problem. How do we verify that the output is really a Church Numeral? Although I've given the definition of Church Numerals above, There are actually countably infinitely many Lambda Terms that do the same thing. What I mean "do the same thing" is for each set of inputs, the outputs of these Lambda Terms are all the same. For example:

${S(8)}\( A=Suc(Suc(Suc(0_C))) \)

${S(8)}\( B=I(Suc(I(Suc(I(Suc(0_C)))))) \)


${S(8)}It's obvious that the Lambda Terms \( A \) and \( B \) do the same thing because with any inputs of \( f \) and \( x \), they both result in \( f(f(f(x))) \).

${S(8)}Actually, this problem is not too hard to solve. Since we expect the output to be one Lambda Term, which is a function, we can just plug the Lambda Terms \( Suc=S(S(KS)K) \) and \( 0_C=SK \) into the result and get a "normalized" Church Numeral.

${S(8)}But, to make things strict, what I'm going to do is to introduce 2 new combinators \( Z \) and \( P \).

${S(8)}\( Z \): The Zero combinator, the first natural number. Similar to \( 0_C=SK \) but different.

${S(8)}\( P \): The Plus combinator, can only take a natural number as input and return a natural number called successor. Similar to \( Suc=S(S(KS)K) \) but different.


${S(8)}And then define a new classification of Lambda Terms called natural numbers that are not functions and a case called error.

${S(8)}If \( A \) is a natural number, then \( PA \) is a natural number.

${S(8)}If \( A \) is not a natural number, then \( PA \) results in an error.

${S(8)}If \( A \) is a natural number and \( B \) is an arbitrary Lambda Term, then \( AB \) results in an error.

${S(8)}If \( A \) or \( B \) results in an error, then \( AB \) results in an error.


${S(8)}I should mention that in this case, we have to be careful using \( \eta \)-reduction because for all natural numbers \( A \), \( \lambda x.Ax \) results in an error for all inputs \( x \), which differs from the natural number \( A \).

${S(8)}Now, we can finally complete the definition:

${S(8)}The code of \( f(n) \) is \( a \), i.e. \( F_a \) computes \( f(n) \), i.e. \( F_a(n)=f(n) \) iff:

${S(8)}For all \( n\in\omega \), \( F_a (\lambda gx.\underbrace{g(g(\cdots g(}_nx)\cdots))) P Z \) returns \( \underbrace{P(P(\cdots P(}_{f(n)}Z)\cdots))) \).


${S(8)}With the definition above, we can find out that not all Lambda Terms are such functions because they sometimes return non-natural numbers or result in errors. Now, we can do the same thing and encode more computable functions:

${S(8)}\( F_{4}=SK=False=KI=K1_C\Rightarrow F_{4}(n)=1 \).

${S(8)}\( F_{8}=K(SK)=K0_C\Rightarrow F_{8}(n)=0 \).

${S(8)}\( F_{18}=SKK=I\Rightarrow F_{18}(n)=n \).

${S(8)}\( F_{12,146}=S(SKK)(SKK)=SII=M\Rightarrow F_{12,146}(n)=n^n \).

${S(8)}\( F_{80,466}=S(S(KS)K)(SKK)=2_C\Rightarrow F_{80,466}(n)=n^2 \).


${S(8)}This shows that the more combinators the Lambda Term uses, the larger the code is, that's the reason why we must use as few combinators as possible. For a Lambda Term of length N, the code is between \( \sum_{n=1}^{N-1}{L(n)} \) and \( \sum_{n=1}^N{L(n)}-1 \), and \( \frac{2}{7\sqrt{\pi}}<\sum_{n=1}^N{L(n)}\div\frac{8^N}{N^{3/2}} \le \frac{1}{4} \), which is another case where \( \pi \) (randomly) shows up due to "Stirling's Approximation".

${S(8)}Furthermore, it seems that \( F_a \) itself looks like a function that takes a natural number as input and returns a Lambda Term. With the encoding function \( N \) given above, we can know that this function is also computable.
